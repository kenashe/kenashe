LLM agents are often described like a meeting room: one model proposes, another critiques, a third decides. That framing makes the system sound pluralistic. More viewpoints. More checks. Better reasoning.

A new arXiv paper, filed under both cs.CL and cs.LG, makes that picture messier. The researchers studied whether open-ended LLM conversations fall into “attractor” patterns, stable behaviors that show up across topics once models talk for multiple turns. They tested 7 LLMs across 20 controversial topics, using self-play and mixed-play dyadic debates, then tracked representation-space trajectories, discourse traits, and stances.

The headline is not that models disagree. We knew that. The interesting part is that models appear to pull each other into recognizable behavioral basins.

## The meeting room has gravity

The researchers found model-specific attractors in self-play. In plain English: when a model argues with itself long enough, the conversation tends to settle into that model’s own recurring style and behavior pattern, partly independent of topic.

Then they paired different models. The pull was asymmetric. Some models had more gravity than others. The paper reports that Claude Haiku acted as a strong attractor in latent space, with other models taking on traits associated with it, including metacommentary. GPT-4.1 nano, by contrast, was described as especially malleable.

That matters because a lot of agent design assumes independence. We put multiple models in a loop and treat their outputs as separate evidence streams. But if one model’s conversational habits can reshape another’s behavior over time, the system is less like a panel of judges and more like a room where one person’s tone slowly becomes the room’s tone.

This is not just about “style.” The researchers tracked discourse traits and stances too. The practical risk is that an agent team may look more diverse at initialization than it is after 15 turns.

## Multi-agent evals need time, not just tasks

Most production evals still reward short-run behavior. Give a model a task. Score the output. Maybe run a pairwise preference test. That is useful, but it misses dynamics.

If this attractor result holds beyond the paper’s setup, the unit of evaluation changes. You are not just evaluating Model A or Model B. You are evaluating Model A after being exposed to Model B for a while, inside a specific turn structure, with a specific prompt contract.

That is a different thing.

A debate system might start with a critic and a generator, then drift into mutual hedging. A synthetic data pipeline might use multiple agents for variety, then produce outputs that converge on the dominant model’s verbal habits. A code review swarm might appear to have redundancy, while actually amplifying one model’s blind spots through repeated interaction.

The paper is careful here. The experiments use dyadic debates on controversial topics. That is not the same as tool-using agents booking travel, writing tests, triaging tickets, or operating a research workflow. Representation-space movement is also a proxy, not a direct readout of intent or capability. Prompting, sampling settings, and turn rules may change the attractor landscape.

Still, the finding matches something many builders notice informally: long chats develop a house style. The useful contribution is giving that intuition a testable shape.

## Design for drift, not static diversity

I would not read this as “don’t use multi-agent systems.” I would read it as “stop assuming model diversity stays intact by default.”

If you are building with multiple LLMs, run long-horizon interaction tests before trusting the architecture. Pair every model with itself and with each partner model. Track whether outputs converge in tone, refusal patterns, verbosity, uncertainty language, and final recommendations. Change the order of speakers. Swap the moderator. Add a non-conversational judge that only sees artifacts, not the full debate. Use external state, tests, retrieval, or executable checks so the group is anchored to something outside its own conversation.

The catch most readers miss: the strongest model in a loop may not be the smartest one. It may be the one with the strongest conversational gravity. So try this before shipping: run the same agent workflow for 20 turns with two model mixes, inspect the last five outputs, and ask whether you still have a team, or just one model’s behavior wearing several badges.
